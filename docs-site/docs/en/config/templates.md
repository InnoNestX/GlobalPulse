# Templates

Templates define the output format for your briefings.

## Template Variables

Available in all templates:

| Variable | Description |
|----------|-------------|
| `{{generatedAt}}` | Generation timestamp |
| `{{timezone}}` | Schedule timezone |
| `{{topicQuery}}` | Topic query string |
| `{{sourceUrl}}` | RSS source URL |
| `{{itemsMarkdown}}` | News items as markdown |
| `{{itemsText}}` | News items as plain text |
| `{{itemsJson}}` | News items as JSON |
| `{{marketReport}}` | AI research output (market mode only) |

## Digest Mode Template

For `reportMode: "digest"`:

```markdown
# 📊 {{topicQuery}} - {{generatedAt}}

## Latest Updates

{{itemsMarkdown}}

---
*Generated by GlobalPulse • {{timezone}}*
```

## Market Mode

In market mode, `{{marketReport}}` contains the full AI-generated research output.

## Provider-Specific Formatting

### Feishu (Markdown)

```markdown
**{{appName}}** 📊

{{itemsMarkdown}}

🕐 {{generatedAt}}
```

### Telegram (HTML)

```html
<b>{{appName}}</b> 📊

{{itemsMarkdown}}

🕐 {{generatedAt}}
```

### Email (HTML)

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>{{appName}}</title>
</head>
<body>
  <h1>{{appName}}</h1>
  <div>{{itemsMarkdown}}</div>
  <footer>
    <p>Generated at {{generatedAt}} ({{timezone}})</p>
  </footer>
</body>
</html>
```

## Custom Variables

Per-schedule variables via Admin UI:

- `{{appName}}` - Configured app name
- `{{scheduleName}}` - Schedule identifier
- `{{reportType}}` - a_share, us_stock, crypto, etc.

## Tips

1. **Keep it concise** - Providers have message limits
2. **Use emojis sparingly** - They render differently across platforms
3. **Test with Preview** - Always use Preview before enabling
4. **Timezone awareness** - Include timezone in footer for clarity
