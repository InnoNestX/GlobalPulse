# 模板

模板定义简报的输出格式。

## 模板变量

所有模板中可用：

| 变量 | 描述 |
|------|------|
| `{{generatedAt}}` | 生成时间戳 |
| `{{timezone}}` | 任务时区 |
| `{{topicQuery}}` | 主题查询字符串 |
| `{{sourceUrl}}` | RSS 源 URL |
| `{{itemsMarkdown}}` | 新闻项的 markdown 格式 |
| `{{itemsText}}` | 新闻项的纯文本格式 |
| `{{itemsJson}}` | 新闻项的 JSON 格式 |
| `{{marketReport}}` | AI 研究输出（仅市场模式） |

## 摘要模式模板

`reportMode: "digest"` 时使用：

```markdown
# 📊 {{topicQuery}} - {{generatedAt}}

## 最新动态

{{itemsMarkdown}}

---
*由 GlobalPulse 生成 • {{timezone}}*
```

## 市场模式

市场模式下，`{{marketReport}}` 包含完整的 AI 研究输出。

## 渠道特定格式

### 飞书 (Markdown)

```markdown
**{{appName}}** 📊

{{itemsMarkdown}}

🕐 {{generatedAt}}
```

### Telegram (HTML)

```html
<b>{{appName}}</b> 📊

{{itemsMarkdown}}

🕐 {{generatedAt}}
```

### Email (HTML)

```html
<!DOCTYPE html>
<html>
<head>
  <meta charset="utf-8">
  <title>{{appName}}</title>
</head>
<body>
  <h1>{{appName}}</h1>
  <div>{{itemsMarkdown}}</div>
  <footer>
    <p>{{generatedAt}} 生成 ({{timezone}})</p>
  </footer>
</body>
</html>
```


## 自定义变量

通过管理后台设置的每任务变量：

- `{{appName}}` - 配置的应用名
- `{{scheduleName}}` - 任务标识符
- `{{reportType}}` - a_share、us_stock、crypto 等

## 技巧

1. **保持简洁** - 渠道有消息长度限制
2. **适度使用 emoji** - 不同平台渲染不同
3. **用预览测试** - 启用前先用预览确认
4. **时区明确** - 底部包含时区便于理解
